# 概述

C++ list 是一种序列容器，对应于双向链表（Doubly Linked List）。与 vector 不同，list 是**节点式存储、离散内存、支持任意位置 O(1) 插入删除**的序列容器。

第一阶段的目标是int类型的list的主要接口，最小实现一个可行的list
第二阶段的目标是进行模板编程、强化异常安全和适配stl
第三阶段是支持自定义分配器
第四阶段是性能优化

# 功能目标

## 离散内存与节点管理

List 不像 vector 那样拥有一整块连续内存，而是由一个个独立的节点（Node）通过指针链接而成。
*   **非连续性**：不支持随机访问（`operator[]`），访问第 n 个元素需要 $O(n)$ 时间。
*   **插入/删除优势**：在已知位置（迭代器指向处）进行插入或删除操作仅需修改指针，无需移动其他元素，时间复杂度为 $O(1)$。
*   **内存开销**：每个元素额外存储两个指针（prev, next），存在一定的空间 `overhead`。

## 迭代器稳定性

List 的一个核心特性是**极强的迭代器稳定性**。
*   插入操作不会导致扩容或内存重分配，因此**所有现存的迭代器、指针和引用都不会失效**。
*   删除操作仅使指向被删除节点的那个迭代器失效。

## 拥有值语义与生命周期管理

同 vector 一样，list 负责内部元素的生命周期。
*   构造时分配节点内存并构造对象。
*   析构时遍历链表，销毁对象并释放节点内存。
*   支持深拷贝和移动语义。

## 支持 STL 生态的接口

虽然不支持随机访问，但 list 提供双向迭代器（Bidirectional Iterator），支持 STL 算法库中除随机访问外的绝大多数算法。

# 实现架构

List 通常实现为**带哨兵节点（Sentinel Node）的循环双向链表**。
*   哨兵节点（通常称为 `head` 或 `end_node`）不存储有效数据。
*   `end()` 迭代器指向这个哨兵节点。
*   空链表时，哨兵的 `next` 和 `prev` 都指向自己。

```
       end() / Sentinel
      ┌───────────────┐
  ┌──➡│   (dummy)     │⬅──┐
  │   └──────┬─▲──────┘   │
  │          │ │          │
  │   ┌──────▼─┴──────┐   │
  └───┤     Node 1    │   │
      └──────┬─▲──────┘   │
             │ │          │
      ┌──────▼─┴──────┐   │
      │     Node 2    │◄──┘
      └───────────────┘
```

## 第一阶段：定义基础结构

### 1. 节点结构 (ListNode)
定义链表的基本单元，包含数据域和指针域。
*   使用结构体模板 `struct ListNode<T>`。
*   包含 `T data`。
*   包含 `ListNode* prev`, `ListNode* next`。

### 2. 迭代器封装 (ListIterator)
List 的迭代器不能像 Vector 那样直接使用原生指针，必须封装为一个类。
*   重载 `operator*`, `operator->` 访问数据。
*   重载 `operator++`, `operator--` 进行移动（指向 `next` 或 `prev`）。
*   重载 `operator==`, `operator!=` 比较相等性。

## 第二阶段：实现主要接口

### 1. 构造与析构
*   `myList()`: 初始化哨兵节点，构建空环。
*   `~myList()`: 遍历释放所有节点，最后释放哨兵。
*   `clear()`: 清空所有数据节点，恢复到初始空环状态。

### 2. 元素访问
*   `front()`: 返回第一个有效节点的数据。
*   `back()`: 返回最后一个有效节点的数据。
*   **注意**：不提供 `operator[]` 和 `at()`。

### 3. 修改器 (Modifiers) (核心优势)
*   `push_back(val)` / `pop_back()`: 尾部操作。
*   `push_front(val)` / `pop_front()`: 头部操作（Vector 不具备的高效操作）。
*   `src/insert(iterator pos, val)`: 在 pos 之前插入。
*   `erase(iterator pos)`: 删除 pos 指向的节点。

### 4. 其它操作 (List Specific)
List 特有的高效率操作（通过修改指针实现，而非拷贝数据）。
*   `splice`: 拼接链表（接合）。
*   `remove` / `remove_if`: 删除满足条件的元素。
*   `unique`: 去重。
*   `merge`: 合并两个有序链表。
*   `sort`: 链表排序（因为 `std::sort` 需要随机迭代器，所以 list 需要自带 `sort`）。

## 第三阶段：内存配置器 (Allocator) 适配

*   使用 `std::allocator` 分配节点内存。
*   使用 `std::allocator_traits` 进行构造和析构。
*   处理 Allocator 的 rebind（因为 `allocator<T>` 需要分配 `ListNode<T>`）。
